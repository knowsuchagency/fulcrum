# 终端管理

Vibora 提供两种终端组织方式：任务终端和持久化终端标签页。

## 任务终端

每个任务都有一个运行在隔离 git 工作树中的终端。这些终端是您运行 Claude Code 和进行工作的地方。

### 任务终端视图

**任务终端** 视图以并行网格形式显示所有任务的终端。这是 Vibora 编排多个 Claude Code 会话的核心功能。

在此视图中您可以：
- 一次查看所有活跃的 Claude 会话
- 点击任意终端进行交互
- 监控多个代理的进度
- 快速识别需要关注的任务

## 持久化终端标签页

对于不适合任务工作树的工作，使用持久化终端标签页。

### 创建标签页

点击终端侧边栏中的 **+** 按钮创建新标签页。标签页可以：
- 命名以便识别
- 按不同用途组织（监控、SSH 会话等）
- 跨重启持久化

### 标签页功能

- **重命名** — 双击标签页名称
- **复制** — 创建当前会话的副本
- **关闭** — 关闭标签页（会话保留直到显式销毁）

## 终端功能

### dtach 会话

终端由 [dtach](https://github.com/crigler/dtach) 会话支持，提供：

- **持久化** — 终端在服务器重启后仍然存在
- **分离/附加** — 断开连接时会话继续运行
- **远程弹性** — 非常适合远程服务器设置

### 滚动历史

终端保持滚动历史。向上滚动查看之前的输出。

## 远程终端

在远程服务器上运行 Vibora 时，即使断开连接，终端也会继续运行。这是 Vibora 进行 AI 代理编排的关键优势。

请参阅 [远程服务器](/zh/guide/remote-server) 了解设置说明。
